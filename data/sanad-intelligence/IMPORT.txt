SANAD Network Intelligence — data import

1. Copy official exports into: data/sanad-intelligence/import/
   - TransactionStatistics.json
   - SanadCenterIncome.json (malformed year keys like "2024.000" are normalized)
   - SanadCenterEmployeesStatistics.json
   - SanadCenterStatistics.json (nested governorate → wilayat → village → count)
   - MostUsedServices.json
   - SanadCenterDirectory.csv (UTF-8; preferred if present) or SanadCenterDirectory.xlsx (first sheet; same bilingual headers)

2. Apply DB migration 0025_sanad_network_intelligence.sql (via your usual drizzle migrate process).

3. Run: pnpm sanad-intel:import
   Requires DATABASE_URL. Optional: SANAD_INTEL_IMPORT_DIR=/absolute/path/to/folder

4. Idempotence: governorate-year metrics for touched years are replaced; workforce & geography tables are cleared and reloaded; directory rows upsert by fingerprint; licence requirement seed is additive by code.

5. Licensing PDF: place at client/public/docs/SanadServicesCenterlicense-Ar.pdf for the compliance page link.

6. Preview / deploy (tRPC 404 "No procedure found"):
   - After git pull, rebuild the client (pnpm build) and redeploy static assets; restart the API process so server/routers.ts is loaded.
   - In DevTools Network, tRPC GET URLs should use procedure paths like "sanad.intelligence.overviewSummary" (nested under sanad). If you still see "sanadIntelligence.overviewSummary", the browser is loading an old bundle — hard refresh (Ctrl+Shift+R) or clear site cache.

7. tRPC 500 / console "Failed query: … sanad_intel_…":
   - The API is reaching MySQL but the SANAD intelligence tables are not there (or the DB user cannot read them). Apply step 2 (migration 0025) on the same database as DATABASE_URL used by the server, then run step 3. After migration, empty tables are OK — overview KPIs may show zeros until import runs.
   - Latest server builds map this case to a clearer tRPC message mentioning migration 0025. If you only see the raw "Failed query" line, restart/redeploy the API so the current server code is running.
